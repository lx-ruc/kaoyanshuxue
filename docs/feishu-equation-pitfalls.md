# 飞书云文档公式排版经验

## 核心规则

1. **禁止 `$$...$$`** — 块级公式语法会被 Markdown 解析器破坏
2. **使用 `<equation>` 标签** — 所有公式（行内+块级）统一用 `<equation>...</equation>`
3. **`<equation>` 内部 `_` 也不安全** — 非单词字符后的 `_` 会被 Markdown 斜体化
4. **遇到 `|_x` 等无法解决的模式时，改用文字标注替代下标**

## 问题详解

### 问题1：`$$...$$` 完全不安全

飞书的 Markdown 解析器先于 LaTeX 处理内容，`$$` 内的以下模式会被破坏：

| LaTeX 模式 | Markdown 误识别 | 后果 |
|-----------|----------------|------|
| `_` 下标 | 斜体标记 `*...*` | 下标内容变成斜体文本 |
| `\right](-1)` | 链接语法 `[text](url)` | `](-1)` 被剥离 |
| `\[...\]` | 链接语法 | 方括号内容丢失 |

### 问题2：`<equation>` 内部 `_` 也不安全

飞书的 Markdown 解析器**先处理 `_`，再识别 `<equation>` 标签**。`<equation>` 内的 `_` 在特定条件下仍会被解释为斜体。

**触发条件**：`_` 前面是非单词字符（`|`, `)`, `+`, 空格等），且同一 `<equation>` 中后面还有可配对的 `_`。

```
# 输入
<equation>\right|_{(n-1)\times(n-1)} = 2D_{n-1}+2</equation>

# 飞书处理后（Markdown 先执行）
*{(n-1)\times(n-1)} = 2D*<equation>...损坏的公式...</equation>
```

**为什么 `A_{ij}` 没问题？** `A` 是单词字符，`A_` 中的 `_` 不触发斜体。只有非单词字符后的 `_`（如 `|_`, `)_`）才会触发。

### 已测试的解决方案及结果

| 方案 | 方法 | 结果 |
|------|------|------|
| `\sb{x}` 代替 `_x` | KaTeX 等价写法 | ❌ 飞书 KaTeX 不支持 `\sb`，显示为无效公式 |
| `\kern0pt_x` | 零宽度间距保护 `_` | ❌ 下标位置偏移到右中，不在右下 |
| `\mathstrut_x` | 不可见占位保护 `_` | ❌ 同上，位置偏移 |
| `\relax_x` | LaTeX 空操作 | ❌ 显示为无效公式 |
| `\_x` | Markdown 转义下划线 | ❌ LaTeX 渲染为字面下划线，非下标 |
| `\underset{x}{\|}` | 下方标注 | ❌ 下标在正下方，不在右下方 |
| **去掉下标，文字标注** | 避开 `_` 冲突 | ✅ **唯一可行方案** |

### 最终方案：文字标注替代问题下标

对于 `|_{...}`, `)_{...}` 等非单词字符后跟下标的模式，去掉下标改为文字标注：

```
# ❌ 会被 Markdown 破坏
<equation>\right|_{(n-1)\times(n-1)} = 2D_{n-1}+2</equation>

# ✅ 去掉下标，用文字标注阶数
<equation>\right| = 2D_{n-1}+2</equation>（其中该行列式为 <equation>(n-1)\times(n-1)</equation> 阶）
```

**何时 `_` 是安全的（不需要处理）**：
- `A_{ij}` — `A` 是单词字符 → 安全
- `D_n` — `D` 是单词字符 → 安全
- `x_{n+1}` — `x` 是单词字符 → 安全

**何时必须去掉下标改为文字**：
- `\right|_x` — `|` 是非单词字符 → 危险
- `)_x` — `)` 是非单词字符 → 危险
- 空格后 `_x` → 危险

## 实际案例

### 案例1：行列式阶数下标（2015年第13题）

```
# 原始 LaTeX（危险）
\right|_{(n-1)\times(n-1)} = 2D_{n-1}+2

# 飞书渲染（损坏）
{(n-1)\times(n-1)} = 2D  ← _ 变成斜体

# 最终修复（文字标注）
\right| = 2D_{n-1}+2  （去掉下标，文字说明阶数）
```

### 案例2：方括号被解析为链接（2016年第13题）

```
# 原始 LaTeX（在 $$ 中）
\left[...\right](-1)^3

# 飞书渲染（损坏）
\left^3  ← ](-1) 被当成链接剥除

# 修复（使用 <equation> 标签）
<equation>\left[...\right]\cdot(-1)^3</equation>  ← 安全
```

## 预处理管线（批量生成时必须）

对于大量题目（如 500+ 题），写入前必须：

1. **公式格式转换**：
   - `$$...$$` → `<equation>...</equation>`
   - 合并被拆分的 `<equation>` 标签
2. **下标安全处理**：
   - 扫描所有 `<equation>` 内容中非单词字符后的 `_`
   - 将 `|_{...}`, `)_{...}` 等模式的下标改为文字标注
   - 保留 `A_{ij}`, `D_n` 等单词字符后的安全下标
3. **OCR 清理**：
   - 数字间距：`1 0 0` → `100`
   - 混合标点统一
4. **写入策略**：
   - `docs +create` + `docs +update --mode append` 分段写入
   - 出问题就 `overwrite` 重写整个文档
   - 不要尝试 `replace_range` 修补损坏的公式

## 增量修复为何不可行

飞书文档中已损坏的公式（`_` 变成 `*...*`、`](` 被剥除）会导致 `replace_range` 文本匹配失败：
- 损坏后的文本与原始 markdown 不同，定位表达式无法匹配
- 多次修复可能引入重复内容
- 5题修复已非常困难，500题的文档几乎不可能增量修复
- 唯一可靠的修复方式是 `overwrite` 重写整个文档
