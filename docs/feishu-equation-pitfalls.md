# 飞书云文档公式排版经验

## 核心规则

1. **禁止 `$$...$$`** — 块级公式语法会被 Markdown 解析器破坏
2. **使用 `<equation>` 标签** — 所有公式（行内+块级）统一用 `<equation>...</equation>`
3. **用 `\sb{}` 代替 `_`** — `<equation>` 标签内部也不安全，`_` 仍会被 Markdown 斜体解析

## 问题详解

### 问题1：`$$...$$` 完全不安全

飞书的 Markdown 解析器先于 LaTeX 处理内容，`$$` 内的以下模式会被破坏：

| LaTeX 模式 | Markdown 误识别 | 后果 |
|-----------|----------------|------|
| `_` 下标 | 斜体标记 `*...*` | 下标内容变成斜体文本 |
| `\right](-1)` | 链接语法 `[text](url)` | `](-1)` 被剥离 |
| `\[...\]` | 链接语法 | 方括号内容丢失 |

### 问题2：`<equation>` 标签内部 `_` 也不安全

飞书的 Markdown 解析器**先处理 `_`，再识别 `<equation>` 标签**。因此 `<equation>` 内的 `_` 仍会被解释为斜体标记。

**触发条件**：`_` 前面是非单词字符（`|`, `)`, `+`, 空格等），后面也有 `_` 可配对。

```
# 输入
<equation>\right|_{(n-1)\times(n-1)} = 2D_{n-1}+2</equation>

# 飞书处理后（Markdown 先执行）
*{(n-1)\times(n-1)} = 2D*<equation>...损坏的公式...</equation>
```

**为什么 `A_{ij}` 没问题？** 因为 `A` 是单词字符，`A_` 中的 `_` 不触发斜体。只有非单词字符后的 `_`（如 `|_`, `)_`）才会触发。

### 解决方案：`\sb{}` 替代 `_`

在 LaTeX/KaTeX 中，`\sb{x}` 等价于 `_x`（下标）。`\sb` 不含 `_`，不会被 Markdown 解析器干扰。

```
# ❌ 危险（_ 可能被 Markdown 斜体化）
<equation>\right|_{(n-1)\times(n-1)} = 2D_{n-1}+2</equation>

# ✅ 安全（\sb{} 完全等效，不含下划线）
<equation>\right|\sb{(n-1)\times(n-1)} = 2D\sb{n-1}+2</equation>
```

**何时必须用 `\sb{}`**：
- `\right|_x` — `|` 是非单词字符 → 必须用 `\sb{x}`
- `)_x` — `)` 是非单词字符 → 必须用 `\sb{x}`
- 空格后 `_x` → 必须用 `\sb{x}`

**何时 `_` 是安全的**：
- `A_{ij}` — `A` 是单词字符 → `_` 不触发斜体，安全
- `D_n` — `D` 是单词字符 → 安全

**最安全的策略**：预处理时将所有 `_` 替换为 `\sb{}`，一劳永逸。

## 实际案例

### 案例1：下标被解析为斜体（2015年第13题）

```
# 原始 LaTeX
\right|_{(n-1)\times(n-1)} = 2D_{n-1}+2

# 飞书渲染（损坏）
{(n-1)\times(n-1)} = 2D  ← _ 变成斜体

# 修复后（使用 \sb{}）
\right|\sb{(n-1)\times(n-1)} = 2D\sb{n-1}+2  ← 正常渲染
```

### 案例2：方括号被解析为链接（2016年第13题）

```
# 原始 LaTeX
\left[4+3\lambda+2\lambda^2\right](-1)^3

# 飞书渲染（损坏）
\left^3  ← ](-1) 被当成链接剥除

# 修复（使用 <equation> 标签即可，] 后紧跟 ( 但不在同一 equation 中重复触发）
```

## 文档创建策略

对于大量题目（如 500+ 题），必须：

1. **预处理管线**：
   - `$$...$$` → `<equation>...</equation>`
   - `_` → `\sb{}` （或仅替换非单词字符后的 `_`）
   - 合并被拆分的 `<equation>` 标签
   - 清理 OCR 数字间距、标点等
2. **一次性写入**：`docs +create` + `docs +update --mode append` 分段写入
3. **出问题就 overwrite**：不要尝试 `replace_range` 修补损坏的公式
4. **模板化生成**：从源数据生成完整 markdown，验证后再上传

## 增量修复为何不可行

飞书文档中已损坏的公式会导致 `replace_range` 文本匹配失败：
- 损坏后的文本与原始 markdown 不同，定位表达式无法匹配
- 多次修复可能引入重复内容
- 5题修复已非常困难，500题的文档几乎不可能增量修复
- 唯一可靠的修复方式是 `overwrite` 重写整个文档

## OCR 质量常见问题

| 问题 | 示例 | 修复 |
|------|------|------|
| 数字间距 | `1 0 0` → `100` | 正则清理 |
| 混合标点 | `（` vs `(` | 统一 |
| 公式拆分 | `<equation>A+</equation><equation>B</equation>` | 合并为单个标签 |
