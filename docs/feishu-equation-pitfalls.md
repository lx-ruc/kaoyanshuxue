# 飞书云文档公式排版经验

## 核心规则：全程使用 `<equation>` 标签，禁止 `$$...$$`

### 问题

飞书的 Lark-flavored Markdown 虽然支持 `$$...$$` 块级公式，但 Markdown 解析器会**先于** LaTeX 处理以下模式，导致公式被破坏：

| LaTeX 模式 | Markdown 误识别 | 后果 |
|-----------|----------------|------|
| `_{...}` 下标中的 `_` | 斜体标记 `*...*` | 下标内容变成斜体文本 |
| `\right](-1)` | 链接语法 `[text](url)` | `](-1)` 被剥离，公式断裂 |
| `\[...\]` | 链接语法 | 方括号内容丢失 |

### 实际案例

**案例1：下标被解析为斜体**

```
# 原始 LaTeX（在 $$ 中）
$$ D_n = 2D_{n-1} + ...|_{(n-1)\times(n-1)} $$

# 飞书实际渲染
{(n-1)\times(n-1)} = 2D   ← 下标 _{} 被当成斜体，变成 *{...}*
```

**案例2：方括号被解析为链接**

```
# 原始 LaTeX（在 $$ 中）
$$ ... \left[4+3\lambda+2\lambda^2\right](-1)^3 $$

# 飞书实际渲染
\left^3   ← ](-1) 被当成链接语法剥除
```

### 解决方案

**所有公式（包括块级公式）统一使用 `<equation>` 标签：**

```
# ❌ 禁止
$$ \begin{aligned} ... \end{aligned} $$

# ✅ 正确
<equation>\begin{aligned} ... \end{aligned}</equation>
```

`<equation>` 标签内的内容不会被 Markdown 解析器处理，LaTeX 语法完全安全。飞书会自动根据内容复杂度（如 `\begin{aligned}`、`\begin{cases}` 等环境）将复杂公式渲染为块级显示。

### 文档创建策略

对于大量题目（如 500+ 题），必须：

1. **预处理**：在上传前将所有 `$$...$$` 转换为 `<equation>...</equation>`
2. **一次性写入**：使用 `docs +create` + `docs +update --mode append` 分段写入
3. **避免增量修复**：如果发现问题，用 `overwrite` 重写整个文档，不要尝试 `replace_range` 修补损坏的公式
4. **模板化生成**：从源数据生成完整 markdown，验证后再上传

### 增量修复的困难

飞书文档中已损坏的公式（`_` 变成 `*...*`、`](` 被剥除）会导致 `replace_range` 的文本匹配失败：
- 损坏后的文本与原始 markdown 不同，定位表达式无法匹配
- 多次修复可能引入重复内容
- 5题修复已非常困难，500题的文档几乎不可能增量修复

### OCR 质量常见问题

| 问题 | 示例 | 修复 |
|------|------|------|
| 数字间距 | `1 0 0` → `100` | 正则清理 |
| 混合标点 | `（` vs `(` | 统一 |
| 公式拆分 | `<equation>A+</equation><equation>B</equation>` | 合并为单个标签 |
